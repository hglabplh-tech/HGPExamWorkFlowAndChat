# Contributing

Thank you for helping improve HGPExamWorkFlowAndChat.

1. Open an issue for substantial behavior or architecture changes.
2. Create a focused branch and include tests for changed behavior.
3. Run `make check` before opening a pull request.
4. Never commit credentials, private keys, student data, exam evidence, model
   artifacts, or production trusted-list caches.
5. Describe security and data-protection effects in the pull request.

By contributing, you agree that your contribution is licensed under the MIT
License and that you will follow the Code of Conduct.

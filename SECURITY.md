# Security Policy

## Supported versions

The project is pre-1.0 alpha software. Only the latest commit on `main` receives
security fixes.

## Reporting a vulnerability

Do not open a public issue for suspected vulnerabilities. Report them privately
to harald.glab-plhak@t-online.de with affected versions, reproduction steps, and
potential impact. Do not include real student records, credentials, private keys,
or examination evidence.

The maintainers will acknowledge a report as soon as practical, investigate it,
and coordinate remediation and disclosure. No deployment should claim eIDAS,
records-retention, accessibility, or educational-regulatory compliance solely
because it uses this source code.
